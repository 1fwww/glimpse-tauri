## Design Context

### Users
Tech workers in productivity settings — developers debugging queries, support agents proofreading replies, PMs annotating screenshots for context, designers marking up references, sales reps rewriting outreach. Comfortable with computers but not necessarily technical. The common thread: they're in the middle of something and don't want to break flow to talk to AI.

**Key user context:** Mid-task, focused, possibly stressed. Glimpse should feel like a calm interruption — helpful without being loud.

### Brand Personality
**Calm, Resourceful, Moderately Playful**

Glimpse is an on-demand companion, not a power tool. It should feel like a quiet, capable assistant that appears when needed and disappears when done. The playfulness comes through in small moments (the eye-wiggle, smooth transitions) — never in the core interaction.

**Tagline:** "Snap it. Ask it. Never lose your flow."

**Logo:** An abstract eye that doubles as a letter "G" — hand-designed, in `#6C63FF` indigo. The eye concept ties to "glimpse" (a quick look) and the screenshot/vision metaphor.

### Aesthetic Direction

**Target feel:** Calm, Light, Smooth, Modern, Humane, Focused-but-not-distracting

**References:**
- iOS liquid glass design language — translucent, layered, soft depth, modern without being cold
- Lark/Feishu screenshot tool — effortless, smooth interactions, invisible complexity
- Anthropic's visual system (claude.ai, brand, docs) — warm neutrals, confident restraint, human-feeling yet polished, sophisticated without being cold
- The feeling of a well-made native macOS utility

**Anti-references:**
- Shottr — too utilitarian, old-school tool aesthetic, lacks warmth
- OpenAI's visual system (chatgpt.com) — too stark, too corporate, lacks warmth and personality
- "Hacker HUD" / cyberpunk aesthetics — too aggressive, alienates non-technical users
- Heavy enterprise UIs — too many borders, too much visual weight
- Generic "AI product" aesthetics — gradient blobs, excessive glow effects

**Theme:** The current dark "Cyber Recon HUD" (cyan #00E5FF + magenta #FF2D78 on deep navy) diverges significantly from the desired direction. The palette and tone are open to evolution toward something warmer, lighter, and more humane.

**Typography:** Outfit (display) + JetBrains Mono (code) — the Outfit choice aligns well with the modern/soft direction. JetBrains Mono serves the technical audience.

### Design Principles

1. **Disappear when not needed.** Glimpse is a utility, not a destination. Every UI element should justify its presence. When the user is done, the tool should vanish cleanly — no lingering chrome, no "are you sure?" friction.

2. **Calm over clever.** Prefer subtle, smooth interactions over flashy ones. No pulsing glows or aggressive animations. Motion should feel physical and natural — things slide, settle, and breathe rather than flash, bounce, or strobe.

3. **Warm, not clinical.** The interface should feel humane and approachable — soft radii, gentle contrast, muted-but-warm tones. Avoid sterile grays and harsh borders. A customer support agent should feel as at home as a developer.

4. **Respect the user's focus.** The user is mid-task. Every design decision should minimize cognitive load: clear hierarchy, obvious actions, no ambiguity about what to do next. The tool serves the task, not itself.

5. **Accessible by default.** Design for color blindness from the start — never rely on color alone to convey meaning. Support reduced motion preferences. Maintain WCAG AA contrast ratios minimum. Accessibility is not a post-MVP concern.

### Accessibility Requirements
- **WCAG AA minimum** across all screens
- **Color blindness safe:** Never use color as the sole indicator of state. Annotation color palette should be distinguishable across common color vision deficiencies (protanopia, deuteranopia, tritanopia)
- **Reduced motion:** Respect `prefers-reduced-motion` — disable pulsing, wiggling, and sliding animations
- **Contrast:** All text must meet 4.5:1 ratio (normal text) or 3:1 (large text) against its background
- **Focus indicators:** Visible keyboard focus states on all interactive elements
